Collectionのダウンロード

Collectionは、モジュール、プラグイン、ロール、プレイブックなどのコンテンツを配布するために、Ansible2.9からフルサポートされた新しい仕組みです。

Ansibleの標準モジュールとは切り離して管理ができるため、Ansibleの新規バージョンのリリースを待たずに、モジュールの修正、更新、名前変更などを行うことができます。

Ansibleの標準モジュールのあるディレクトリは、ansible --versionで確認できます。


$ ansible --version
ansible 2.9.9
  config file = /home/ansibleuser/.ansible.cfg
  configured module search path = ['/home/ansibleuser/.ansible/plugins/modules', '/usr/share/ansible/plugins/modules']
  ansible python module location = /home/ansibleuser/.local/lib/python3.6/site-packages/ansible
  executable location = /home/ansibleuser/.local/bin/ansible
  python version = 3.6.9 (default, Apr 18 2020, 01:56:04) [GCC 8.4.0]

Ansibleモジュール

    Ansible 2.8より以前のFortiGate用標準モジュール(Ansibleインストール時に入るモジュール)は、ごく限られた設定項目しかカバーしていませんでした。

    標準モジュールでカバーしていない項目については、fortiosconfigモジュールを使って設定が可能です。

    このfortiosconfigモジュールは、Githubで公開していますが、別途fortiosapi(pythonライブラリ)が必要です。

    Ansible 2.8以降、標準モジュールを大幅に追加し、2.9ではすべての設定項目をカバーしていますので、これからAnsibleの利用を考えている方は、2.9が推奨です。

    Ansible 2.8以降では、更にhttpapi(Ansibleプラグイン)に対応していますので、fortiosapi(pythonライブラリ)のインストールが不要です。

    また、fortiosapiではできなかった、インベントリファイルの利用が可能になるため、ぜひAnsible 2.9をご利用ください。

Collectionを活用する

    Collectionは、モジュール、プラグイン、ロール、プレイブックなどのコンテンツを配布するための新たな仕組みです。

    Ansibleの標準モジュールとは切り離して管理ができるため、Ansibleの新規バージョンのリリースを待たずに、モジュールの修正、更新、名前変更などを行うことができます。

    CollectionはAnsible 2.9でフルサポートされるようになり、FortiGate用のCollectionもすでに公開しています。

    https://galaxy.ansible.com/fortinet/fortios

    https://github.com/fortinet-ansible-dev/ansible-galaxy-fortios-collection

    Collectionのダウンロードは、ansible-galaxyコマンド、もしくは上記のgalaxyページからtar形式でダウンロードできます。

         $ ansible-galaxy collection install fortinet.fortios
